# mao-decide 分析示例

以下示例展示了完整的分析流程和输出标准。注意两个示例的区别：示例一中用户的认知有偏差，示例二中用户的判断是准确的。

---

## 示例一：要不要辞职创业（认知纠偏型）

**用户输入：** 我在考虑要不要辞职创业，做一个AI教育产品

### 调查研究阶段（提问摘录）

Q1: 你现在工作的具体不满是什么？不是"不开心"，是具体哪些事让你想走？
> 用户：工资三年没涨，技术栈老旧，领导不重视我的方案

Q2: AI教育产品的方向想到什么程度了？有没有验证过市场需求？
> 用户：有个大概想法，想做面向中小学的AI编程教育，还没验证

Q3: 经济状况如何？辞职后能支撑多久没收入？
> 用户：有大概20万存款，在二线城市，月开销8000左右

Q4: 家人什么态度？
> 用户：老婆支持但有点担心，父母反对

Q5: 你说领导不重视你的方案，具体是什么方案？和你想创业的方向有关系吗？
> 用户：我提过在公司内部做AI教育方向的产品线，被否了

### 输出

```
═══ mao-decide 分析报告 ═══

【决策问题】要不要辞职创业做AI教育产品

── 调查研究 ──
• 客观事实：
  - 当前工作三年未涨薪，技术栈老旧
  - 曾在公司内部提议AI教育产品线，被否决
  - 创业方向：面向中小学的AI编程教育，尚未市场验证
  - 存款20万，月开销8000，可支撑约25个月
  - 配偶支持但有顾虑，父母反对

• 各方立场：
  - 公司：不看好AI教育方向（至少在当前阶段）
  - 配偶：支持但需要安全感
  - 父母：反对（具体原因未深入了解）

• 已剥离的情绪/假设：
  - "领导不重视我"——这更像是一种受挫感。事实是：你的AI教育提案被否了，但这不等于你整个人不被重视
  - "老婆支持"——支持的程度需要具体化：是支持你探索，还是支持你明天就辞职？

• 信息盲区：
  - 公司否决AI教育提案的具体原因是什么？是方向不对还是时机不对？
  - 中小学AI编程教育的市场竞争格局（编程猫、核桃编程等已有玩家）
  - 你的核心竞争力是什么？技术能力还是教育资源？

── 矛盾分析 ──
★ 主要矛盾：不是"打工vs创业"，而是"你想做AI教育这件事"与"你还没有验证这件事能不能做成"之间的矛盾
  └ 主要方面：市场验证的缺失是决定性的。你对AI教育的热情是真实的，但热情不等于市场机会。
  └ 来源启发：这个思路来自毛泽东在《实践论》中的核心观点——认识必须经过实践检验。你现在的"认识"（AI教育有机会）还没有经过"实践"（市场验证）的检验，就急着做最大的决策（辞职），这是把认识的第一阶段当成了最终结论。

• 次要矛盾1：当前工作的成长瓶颈——真实存在，但不是辞职的充分理由
• 次要矛盾2：家庭期望的分歧——父母的反对需要了解具体原因，可能包含有价值的信息

⚡ 认知校准：你把这个决策框架设定为"辞职vs不辞职"，但真正的问题是"AI教育这个方向到底成不成立"。如果方向成立，辞职是自然的下一步；如果方向不成立，辞职就是冲动。你跳过了最关键的一步。

── 实践策略 ──
▶ 明确判断：现在不应该辞职。不是因为创业不好，而是因为你还没有做最基本的市场验证。在没有验证的情况下辞职，不是勇气，是赌博。

• 具体行动：
  1. 用接下来2个月的业余时间做市场调研：找10个中小学家长深度访谈，了解他们对AI编程教育的真实需求和付费意愿
  2. 研究现有竞品（编程猫、核桃编程、童程童美），找到它们没覆盖到的需求缺口
  3. 做一个最小可行产品（哪怕是一个微信群+手动教学），验证有没有人愿意付费
  4. 搞清楚公司否决你提案的具体原因——如果是"方向不对"，这是重要的反面信息

• 可团结的力量：
  - 配偶：让她参与市场调研过程，既是支持也是共同决策
  - 公司否决你提案的那个领导：找他聊聊具体原因，他的反对意见可能是免费的市场分析

• 战术要点：
  - 战略上：你想做AI教育的方向感是对的，教育+AI确实是大趋势
  - 战术上：但"大趋势"不等于"你现在做这个具体产品能成"。趋势对了，切入点错了，一样会失败

⚠ 需持续观察：
  - 如果2个月调研后发现确实有未被满足的需求，且你能提供差异化价值，那时候再认真考虑辞职
  - 如果调研发现市场已经很卷，没有明显缺口，那就需要重新审视方向

📖 推荐阅读：
  - 《实践论》——理解"实践-认识-再实践"的螺旋，你现在需要的是第一轮实践
  - 《反对本本主义》——"没有调查就没有发言权"，对创业同样适用
```

## 示例二：要不要接受降薪的管理岗（确认用户判断型）

**用户输入：** 公司给我一个管理岗的机会，但要降薪15%，从技术转管理。我觉得应该接，但家里人都说我傻。

### 调查研究阶段（提问摘录）

Q1: 这个管理岗具体管什么？团队多大？
> 用户：带一个8人的后端团队，负责公司核心产品线

Q2: 降薪15%是暂时的还是永久的？有没有绩效回升的机制？
> 用户：试用期6个月降15%，之后根据团队绩效调整，上限比现在高30%

Q3: 你为什么觉得应该接？具体原因是什么？
> 用户：我做了6年纯技术，感觉到了天花板。而且这个产品线是公司未来3年的重点，管理岗能接触到更多资源和决策层

Q4: 家人反对的具体理由是什么？
> 用户：主要是觉得降薪不合理，"升职还降薪，肯定是坑"

### 输出

```
═══ mao-decide 分析报告 ═══

【决策问题】要不要接受降薪15%的管理岗位

── 调查研究 ──
• 客观事实：
  - 从高级工程师转为8人后端团队负责人
  - 负责公司核心产品线（公司未来3年重点）
  - 薪资：试用期6个月降15%，之后上限比现在高30%
  - 技术岗已做6年，感到天花板
  - 家人反对，理由是"升职降薪不合理"

• 各方立场：
  - 公司：愿意给核心产品线的管理权，降薪是试用期机制
  - 家人：从短期收入角度反对
  - 用户本人：倾向接受，看重长期发展

• 已剥离的情绪/假设：
  - 家人的"肯定是坑"是基于"升职必须加薪"的假设，不是基于对这个具体岗位的分析

• 信息盲区：
  - 试用期考核的具体标准是什么？
  - 如果试用期没通过，能不能回原岗位？

── 矛盾分析 ──
★ 主要矛盾：技术天花板的突破需求 与 短期收入损失 之间的矛盾
  └ 主要方面：技术天花板是决定性的。6年纯技术到了瓶颈，这个判断大概率是准确的——后端工程师的纯技术路线在大多数公司确实有明确的天花板。
  └ 来源启发：这个局面类似毛泽东在《论持久战》中分析的"战略相持阶段"——你在技术路线上已经进入相持，继续耗下去不会有质变，需要开辟新的战场。

• 次要矛盾：家庭期望管理——家人的反对基于不完整的信息，不构成实质性障碍

⚡ 你的判断是准确的。这不是"升职降薪"的坑，而是一个用6个月15%的薪资差换取职业赛道切换的机会。核心产品线的管理岗意味着接触决策层和资源分配，这是纯技术岗无法获得的。

── 实践策略 ──
▶ 明确判断：接。这是一个值得付出短期代价的战略性机会。

• 具体行动：
  1. 接受offer前，确认两件事：试用期考核标准是什么？没通过的退路是什么？
  2. 和家人沟通时，把账算清楚：6个月降15%的总损失是X元，但之后上限高30%，一年内就能回本
  3. 上任后前3个月，重点建立团队信任，不要急于推技术改革

• 可团结的力量：
  - 团队中的技术骨干：你的技术背景是管理这个团队的天然优势
  - 给你这个机会的上级：他选你说明认可你，这是你的后盾

• 战术要点：
  - 战略上：从技术到管理是打开职业天花板的正确方向
  - 战术上：管理岗的前6个月是最危险的，重点是"站稳"而不是"出彩"

⚠ 需持续观察：
  - 试用期考核标准如果模糊，要主动和上级对齐预期
  - 如果发现这个"核心产品线"实际上是个烫手山芋（资源不到位、目标不合理），要及时评估

📖 推荐阅读：
  - 《论持久战》——理解战略相持阶段为什么需要主动寻求突破
  - 《星星之火，可以燎原》——在新领域起步时如何保持信心
```
